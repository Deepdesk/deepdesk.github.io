Deepdesk Chrome Extension integration

Introduction

The Deepdesk Chrome Extension empowers agents with AI by giving real-time suggestions above the agent's input field during a conversation.

In order to function properly the Deepdesk Chrome Extension needs the following information:

  1. Reference to the input field
  2. Conversation ID (a.k.a. Case ID)
  3. Notification when the typed message is sent

Although all information can be derived from the DOM, Events, window or location, without a proper "contract" this strategy is vulnerable to changes.

Below we describe several approaches to make to Deepdesk Chrome Extension integration more solid.

Reference to input element and a conversation ID

In order to provide the Deepdesk Chrome Extension with a reference to the input element and a conversation ID we suggest providing the following data attribute on the agents input element containing the conversation ID: 

data-autocomplete-conversation-id="XXXXXXXX"

The Deepdesk Chrome Extension can detect when an element with the data attribute is added to the page and attach itself to it and provide suggestings in the context of the provided conversation ID.

Detecting an inserted input element (textarea/contenteditable div) is trivial for the Deepdesk Chrome Extension using the strategy described here. You can also inspect the source code of the proof of concept below.

Reference to input element and a conversation ID (alt)


    

        An alternative method to provide the Deepdesk Chrome Extension with a reference to the input element and to provide the conversation ID is to utilize Custom DOM events.
    


    

        When a conversation is done rendering in the DOM, we suggest the platform to emit a CustomEvent named 'ConversationLoaded' on the document with the following payload:

        
const event = new CustomEvent('ConversationLoaded', {
    conversationId: "XXXXXXXX", // conversation ID
    inputElementId: "YYYY", // element id of the textarea or contenteditable div
});
document.dispatchEvent(event);


    


    

        The Deepdesk Chrome Extension can listen to the 'ConversationLoaded' event and attach itself to the provided input element and provide suggestings in the context of the provided conversation ID.
    


    

        See Creating and triggering events for details.

Notify when the typed message is sent


    

        We propose utilizing DOM events to notify the Deepdesk Chrome Extension when the submit process is started and to notify the Deepdesk Chrome Extension when the message is succesfully sent.
        These events have to be emitted from the input element itself.

        
/* Agent submits message by Enter or by clicking the submit button: */
textareaElement.dispatchEvent(new Event('ConversationMessageSubmit'));

/* Platform does stuff */

/* When message is sent successfully: */
textareaElement.dispatchEvent(new Event('ConversationMessageSent'));
